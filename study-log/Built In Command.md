쉘 자체에 포함된 커맨드

- type `명령어`
- man 에서, `SHELL BUILTIN COMMANDS` 섹션에서 확인 가능

> 즉, 경로들을 찾을 필요없이 쉘이 바로 실행해줘야 하는 명령어들인거 같다...?
> -> 이에따라, 적절히 전략이 수립되어야 한다.
> Built In 커맨드면 바로 실행, 아니면 찾아서 실행

### 실행 커맨드

- `:` : 아무것도 수행하지 않고, 종료 코드 0을 반환
- `exec command` : 현재 쉘이 지정된 command 로 치환되어 실행 (즉, 실행하고 바로 종료)
- `source` : 쉘 스크립트 내 다른 쉘 스크립트 파일 불러와서(포함) 실행하기

### 쉘 종료, 반환

- `exit` : 쉘 종료
- `return` : n 을 반환하고, 쉘 함수에서 빠져나옴

### 현재 작업 디렉토리 확인 및 변경

- pwd : 현재 디렉토리 출력
- cd : 현재 디렉토리 변경 ( 지정된 디렉토리로 이동 )

### 기타

- `echo [args]` : 인자 args 를 터미널에 출력함
- `eval [args]` : 인자 내 변수로부터 참조해 값을 추출
- `export` : 쉘 변수를 환경 변수로 선언
- `kill [PID]` : 지정 프로세스를 중지 - 정확히는 프로세스에 SIGTERM 시그널을 송신
- `read [var]` : 표준 입력에서 1행 읽고, 변수에 저장
- `type [명령어]` : 명령어 유형 보여줌
- `unset` : 지정된 쉘 변수, 쉘 함수 삭제
